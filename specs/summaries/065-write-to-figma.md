# 065-write-to-figma

## Status
Not Implemented

## What it proposes
Add write capabilities to Figma canvas (creating/modifying nodes and properties) by bridging the remote MCP server to a Figma plugin via WebSocket. Since Figma's REST API cannot modify canvas elements, a custom Figma plugin running inside Figma Desktop/Web would receive commands from the server and execute them via the `figma.*` Plugin API.

## Architectural decisions made
- Use a Figma plugin (to be built) that connects to the MCP server via WebSocket (`wss://`) — Figma plugins can connect to remote domains, not just localhost
- Pattern A (Direct WebSocket) recommended as primary: lowest latency (~50ms), true bidirectional communication
- Authentication via a 6-character pairing code generated by the plugin; user enters code in MCP client to link their MCP session to the plugin connection
- Five patterns analyzed (A: Direct WebSocket, B: SSE+HTTP, C: HTTP Polling, D: REST API Hybrid, E: Local Relay) — Pattern A + Key Pairing selected
- Figma REST API used only for read operations and comments; canvas writes must go through plugin

## What still needs implementing
- Figma plugin (UI + code.js worker) with WebSocket client and `figma.*` write capabilities
- `/figma-bridge` WebSocket endpoint on the server to relay commands between MCP sessions and plugin connections
- Pairing code generation, registration, and session-linking logic on the server
- "Connect to Figma" auth option in the MCP OAuth/auth flow for users to enter the pairing code
- MCP tools that send write commands through the bridge (create node, update property, etc.)
- JWT/token handling for the plugin connection (scoping plugin access to the correct user session)
